## Implementation Notes

### Goal

v2.8 Agile Loop の orchestrate 統合フェーズ。A2a で実装した cycle-retrospective skill を orchestrate の Block 2f として組み込み、pre-commit-gate.sh の retro_status check を有効化する。self-dogfood として A2b 自身の Cycle doc に cycle-retrospective を手動実行してから commit する。

### Design Approach

#### orchestrate/SKILL.md 圧縮戦略

現在 106 行。Block 2f 挿入（約 3-4 行）前に 100 行以下に圧縮する必要がある。
圧縮対象:
- Block 2c の `rm -f` コードブロック → インライン 1 行
- Block 2a-2d の冗長なコード例 → インライン化または参照リンク化
- 目標: 96 行以下（Block 2f 挿入後に ≤ 100 行）

#### Block 2f 位置

```
Block 2e: DISCOVERED — スコープ外項目を issue 起票
Block 2f: RETROSPECTIVE — Skill(dev-crew:cycle-retrospective) 実行
  - abort シグナル受信時: COMMIT をスキップしてユーザーに報告
  - proceed / skip (retro_status != none) は Block 3 へ自動進行
Block 3: Finalization (COMMIT)
```

#### pre-commit-gate.sh retro_status check

```bash
# retro_status check (A2b で有効化)
retro=$(awk '/^---$/{c++;next} c==1{print}' "$ACTIVE_CYCLE" | grep '^retro_status:' | head -1 | sed 's/^retro_status: *//' || true)
if [ -z "$retro" ]; then
  echo "WARN: retro_status field not found (legacy Cycle doc). Skipping check."
elif [ "$retro" = "none" ]; then
  echo "BLOCK: retro_status is 'none'. Run cycle-retrospective before commit."
  exit 1
fi
# captured or resolved: PASS
```

#### self-dogfood 手順

1. A2b TDD サイクル完了 (REVIEW まで)
2. `Skill(dev-crew:cycle-retrospective)` を本 Cycle doc に対して手動実行
3. retro_status が `captured` または `resolved` に遷移したことを確認
4. `bash scripts/gates/pre-commit-gate.sh` で PASS 確認
5. COMMIT

#### TC-07/TC-09 の 2>&1 パターン

stdout/stderr 結合が必要な gate スクリプトのテストでは `$(cmd 2>&1)` で出力キャプチャ。
