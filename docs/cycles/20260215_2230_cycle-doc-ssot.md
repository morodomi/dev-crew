## Implementation Notes

### PLAN Section

#### Background
Issue #16 combines two token optimization strategies:
- **C案 (Cycle doc SSOT)**: phase-compact writes structured Phase Summary to Cycle doc; subagents restore context from Cycle doc instead of conversation history
- **A案 (Hybrid delegation)**: orchestrate evaluates phase output size and chooses between PdM direct execution or subagent delegation

This approach reduces token consumption in orchestrate by:
1. Eliminating conversation history repetition across phases
2. Selectively avoiding subagent spawn overhead for lightweight phases

#### Design Decisions

1. **Phase Summary Structure Enhancement** (phase-compact):
   - Extend existing Phase Summary format (Artifacts/Decisions/Next Phase Input) with standardized fields
   - Add structured data for delegation decision: line count, file count, test count
   - Maintain backward compatibility with v5.0 Phase Summary format

2. **Subagent Context Initialization** (orchestrate steps-*.md):
   - Block 0 (Prerequisite Check) already exists (#15), no modification needed
   - Add explicit Cycle doc path in subagent spawn prompt
   - Format: `"Cycle doc: [path]. [フェーズ指示]"`

3. **Hybrid Delegation Logic** (orchestrate):
   - Add decision point after each phase completion
   - Evaluate Phase Summary metrics against threshold
   - Decision table in reference.md

4. **Default Behavior**:
   - INIT, COMMIT → always PdM direct (no delegation)
   - REVIEW → always delegate (quality-gate complexity requires subagent)
   - Others → default to delegation, override if previous phase is lightweight

5. **Validation**:
   - New test script to validate Phase Summary structure
   - Ensure structured fields are present and parseable

#### Test Strategy
- Structure validation: bash script checking Phase Summary format
- Regression: existing test-phase-compact.sh ensures backward compatibility
- Cross-reference: test-cross-references.sh covers orchestrate/phase-compact link integrity
